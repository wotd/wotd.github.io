Diagrams as a Code

April 17, 2023

Introduction#

Recently I wanted draw simple network diagram and I recalled nice application I know. It can be used to draw cloud diagrams and it supports main Cloud providers like Azure, AWS, GCP. Just take a look on webpage where you can find some nice examples.

I’ve prepared simple Docker image so you can try application without installing anything.

Usage#

Diagrams webpage is full of nice examples and documentation, so go there for furhter reading. If you know python :snake: you are good to go :smile: Let’s see sample provided by developer:

# diag.py
from diagrams import Diagram
from diagrams.aws.compute import EC2
from diagrams.aws.database import RDS
from diagrams.aws.network import ELB

with Diagram("Web Service", show=False):
    ELB("lb") >> EC2("web") >> RDS("userdb")

Start with importing Diagram class, EC2, RDS and ELB. We can add diagrams title Diagram("Web Service", show=False). Parameter show=False disables autoopening of new images (I am using docker anyway). Other nice parameter is outformat which can be used to specify output file format (png (default), jpg, svg, pdf and dot).

Docker#

This is simple Dockerfile I used to build image:

FROM alpine:3.17.3

RUN mkdir /src
WORKDIR /src
# Install dependencies
RUN apk add py3-pip gcc python3-dev musl-dev graphviz fontconfig \
    # Install fonts
    && apk add font-inconsolata font-dejavu font-noto font-noto-cjk font-awesome font-noto-extra \
    && pip install diagrams

CMD [ "diagram.py" ]
ENTRYPOINT [ "python" ]

I am using alpine linux, set /src as a WORKDIR. Next, I am installing dependencies and fonts required by Diagrams.

ENTRYPOINT [ "python" ] - python interpreter as a default application running after docker image start. CMD [ "diagram.py" ] - argument for ENTRYPOINT - default name of file with diagram code.

Now we can build image using well known command:

docker build . -t diagrams

Usage#

To create your own diagram using Docker image run command within directory with diagram code. We will mount current dir into docker:

docker run --rm -v ${PWD}:/src diagrams diag.py

:boom::boom::boom: Image will be in the same directory.

Happy diagramming :smile:
